Language commands - how to define tests

The languages provides you with several commands. In this section I will introduce these commands and explain their meaning.

  • create_test command
  • single_choice_question command
  • multiple_choice_question command
  • open_answer_question command
  • pairing_question command
  • run_test command

Every command has to be written on a new line, and it has to be ended with a semicolon.

create_test command

create_test command is used to create a test. This command is used in a single file only once (right after the require_relative clause) - it creates a single test (definition of multiple tests in a single file is not supported). Example of this command is presented in following figure. You type create_test and after that you list its parameters delimited by commas - first parameter is the test title, and the second one is the minimal points for successful test pass.

After this command you use following commands to create a list of questions that belong to the test.

In this question type you can provide only a single correct option. Moreover, the question has to have also at least a one incorrect option. Options for answers are defined using following commands:

correct_answer command defines a correct answer/option for the given question.

incorrect_answer command defines an incorrect answer/option.

Notice that the answer commands are indented from the question. While it is not necessary, it makes clear that answers are subordinate to the question - it indicates that they belong to that question. I would recommend you to do the same, since it can make it easier for you to read your own test definition.

multiple_choice_question command

multiple_choice_question command serves for creating a question with multiple correct answers (see example in the following figure, the question uses checkboxes to allow selection of multiple answers):

The command is used as follows (first parameter defines the text of the question, the second parameter defines the number of points for the question):

In this question type you have to provide at least one correct answer option, and at leat one incorrect answer option. Points are awarded according to the following system. For each correctly selected answer you are awarded a proprotion of the full points available for the given question (e.g., if there are 9 points for the question, and there are 3 correct answers, each correct answer is worth 3 points). If you select also an incorrect answer (even only one), you will get 0 points.

pairing_question command

pairing_question command serves for creating a question in which the tested subject has to correctly pair up provided keywords (as you can see in the following example, for each value on the left you have to find a correct value on the right):

In this question you do not define correct or incorrect answer options - you use pair command to provide correct pairs that belong together. The language mixes them to provide randomness in the question presentation.

Points for this question type are awarded as follows. For each correctly paired up pair you get a portion of the points. In our example there are 4 pairs for 20 points together, that means 5 points for each pair.

run_test command

run_test command ends the test definition and serves for running a defined test (without it, running the project will not result in the browser opening the test). The command also checks for any possible errors to make sure the definition is correct and consistent. There are no parameters to the command.
